## Code of Conduct

This project and everyone participating in it is governed by the [Surrogate Key Trim Code of Conduct](CODE_OF_CONDUCT.md). By participating, you are expected to uphold this code.

## Getting Started

Contributions to Surrogate Key Trim should focus on enhancing the core functionality of the generate_surrogate_key macro with the trim parameter. 

### Prerequisites

- dbt (1.0.0 or higher)

### Local Development Setup

1. Fork the repository
2. Clone your fork
3. Create a new branch for your feature or bug fix

## Development Process

### Testing

Before submitting a pull request, please ensure that:
- Your changes don't break existing functionality
- You've tested your changes with different dbt adapters if applicable
- You've documented any new functionality

### Code Style

- Follow dbt macro best practices
- Maintain backward compatibility
- Use clear, descriptive names for variables and parameters

## Pull Request Process

1. Update the README.md with details of changes if needed
2. Add tests for the new functionality if applicable
3. Ensure the code follows dbt best practices
4. Submit your pull request to the main branch

## Questions?

If you have any questions about contributing, feel free to open an issue.
